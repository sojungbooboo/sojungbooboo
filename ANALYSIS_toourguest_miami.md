# toourguest.com/preview/miami 디자인 분석 보고서

## 📋 개요
toourguest.com의 "Miami" 테마 모바일 청첩장 페이지의 디자인 스타일, 레이아웃, 애니메이션, UI/UX 패턴을 분석한 문서입니다.

---

## 1. 전체 디자인 컨셉 & 분위기

### 1.1 컬러 팔레트
- **주요 색상**: 부드러운 파스텔 톤 (핑크, 베이지, 화이트 계열)
- **강조 색상**: 금박/메탈릭 느낌의 골드 또는 브론즈 포인트 컬러
- **배경**: 밝고 깔끔한 화이트 베이스 또는 그라데이션 배경
- **텍스트**: 높은 대비를 위한 다크 그레이 또는 블랙

### 1.2 타이포그래피
- **메인 타이틀**: 큰 글자, 세리프 또는 세련된 산세리프 폰트
  - 웨딩 날짜, 커플 이름에 사용
  - 화면 중심 또는 상단 배치
- **본문**: 얇거나 보통 굵기의 산세리프
  - 행간(leading) 여유 있게 설정
  - 중앙 정렬(center-aligned) 또는 좌측 정렬
- **영문**: 세련된 영어 폰트 (Allura, Playfair Display 등 스타일)

### 1.3 시각적 요소
- **장식 요소**: 미니멀한 장식 (꽃잎, 라인, 심플한 도형)
- **그래픽**: 아이콘 중심의 심플한 디자인
  - 달력, 위치, 갤러리 아이콘 등
- **이미지 처리**: 
  - 필터 효과 (오버레이, 블러)
  - 풀스크린 또는 카드 형태의 이미지 배치

### 2.2 각 컴포넌트 상세 분석

#### 2.2.1 인트로 / 커버 화면
**특징:**
- 전체 화면 높이 (100vh) 사용
- 배경: 풀스크린 이미지 또는 비디오
- 텍스트 오버레이: 반투명 배경 또는 텍스트 쉐도우로 가독성 확보
- 애니메이션: 페이드 인, 슬라이드 업 효과
- 인디케이터: 스크롤 유도 애니메이션 (화살표 아래로 움직임)

**구현 아이디어:**
```css
.cover-section {
  height: 100vh;
  display: flex;
  flex-direction: column;
  justify-content: center;
  align-items: center;
  position: relative;
  background-image: url(...);
  background-size: cover;
  background-position: center;
}

.cover-section::before {
  content: '';
  position: absolute;
  top: 0;
  left: 0;
  right: 0;
  bottom: 0;
  background: rgba(255, 255, 255, 0.3); /* 오버레이 */
}
```

**특징:**
- 카드 형태의 섹션

- 그림자 효과로 깊이감 (box-shadow)

#### 2.2.3 결혼식 일정 (Calendar)

#### 2.2.4 사진 갤러리 (Gallery)
**특징:**
- **레이아웃 옵션 1**: 그리드 형태의 썸네일
  - 2열 또는 3열 그리드
  - 클릭 시 모달 또는 전체 화면 뷰어
  
- **레이아웃 옵션 2**: 가로 스크롤 슬라이더
  - 스와이프 제스처 지원
  - 좌우 네비게이션 버튼

**애니메이션:**
- 이미지 로딩 시 페이드 인
- 슬라이드 전환 효과 (transform: translateX)
- 확대/축소 제스처 (pinch zoom)

#### 2.2.5 오시는 길 (Location)
**특징:**
- 지도 컴포넌트 (Google Maps 또는 Kakao Map API)
- 주소 정보
- 교통 수단 아이콘 (자동차, 지하철, 버스)

**UI 패턴:**

- 아이콘과 텍스트 조합

#### 2.2.7 방명록 (Guestbook)
**특징:**
- 간단한 입력 폼 (이름, 메시지)

- 스크롤 가능한 메시지 리스트

#### 2.2.8 공유 버튼 (Share)

## 3. 애니메이션 & 인터랙션

### 3.1 페이지 진입 애니메이션
- **페이드 인 (Fade In)**: 전체 페이지가 서서히 나타남
- **슬라이드 업 (Slide Up)**: 하단에서 상단으로 이동하며 등장

```css
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(30px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```

### 3.2 스크롤 기반 애니메이션 (Scroll Reveal)
- 섹션이 뷰포트에 들어올 때 애니메이션 실행
- **Intersection Observer API** 활용

```javascript
// 예시: 섹션이 보이면 페이드 인
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('fade-in-active');
    }
  });
});
```

### 3.3 호버/터치 인터랙션
- **버튼**: 호버 시 약간의 스케일 증가 (transform: scale(1.05))
- **카드**: 그림자 강화로 떠오르는 효과
- **이미지**: 클릭/터치 시 확대 애니메이션

### 3.4 제스처 인터랙션
- **스와이프**: 갤러리 이미지 좌우 스와이프
- **풀 투 리프레시**: (선택 사항) 페이지 새로고침
- **스크롤**: 부드러운 스크롤 (smooth scroll)

### 3.5 전환 효과 (Transitions)
- **섹션 간 전환**: 부드러운 스크롤
- **모달 열기/닫기**: 페이드 + 스케일 효과
- **이미지 변경**: 크로스페이드 또는 슬라이드

## 4. UI/UX 패턴

### 4.1 모바일 우선 디자인 (Mobile First)
- **뷰포트**: 최대 너비 제한 (예: 500px)
- **터치 친화적**: 최소 터치 영역 44x44px
- **세로 스크롤**: 가로 스크롤 최소화
- **반응형**: 데스크톱에서는 중앙 정렬된 카드 형태

### 4.2 사용자 가이던스
- **스크롤 인디케이터**: 초기 화면에서 아래로 스크롤하라는 시각적 힌트
- **섹션 제목**: 명확한 섹션 구분
- **액션 버튼**: 명확한 CTA (Call To Action) 버튼

### 4.3 정보 계층 구조
- **주요 정보 우선**: 날짜, 장소, 이름 등 핵심 정보 강조
- **부가 정보 후순위**: 축의금, 방명록 등은 하단 배치
- **시각적 강조**: 폰트 크기, 색상, 여백으로 중요도 표현

### 4.4 접근성 (Accessibility)
- **텍스트 가독성**: 배경과 텍스트의 명도 대비 충분
- **터치 영역**: 버튼/링크 충분한 크기
- **로딩 상태**: 이미지 로딩 중 플레이스홀더 표시

### 4.5 성능 최적화
- **이미지 최적화**: WebP 포맷, 적절한 압축
- **Lazy Loading**: 뷰포트 밖 이미지는 지연 로딩
- **코드 스플리팅**: 필요 시 컴포넌트별 코드 분리

## 5. 기술적 구현 참고사항

### 5.1 CSS 기법
- **Flexbox/Grid**: 레이아웃 구성

- **Transform/Transition**: 애니메이션

- **Intersection Observer**: 스크롤 애니메이션
- **useState/useEffect**: 상태 관리
- **React Router** (필요시): 여러 페이지 구성
- **Context API**: 전역 상태 (테마, 모달 등)

### 5.3 외부 라이브러리 추천
- **이미지 갤러리**: Swiper.js, react-image-gallery
- **애니메이션**: Framer Motion, react-spring
- **지도**: react-kakao-maps-sdk, @react-google-maps/api
- **공유 기능**: 카카오 SDK (kakao.min.js)
